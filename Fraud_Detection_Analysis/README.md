# Financial Transaction Fraud Detection Analysis

## Overview
Financial institutions process millions of transactions every day. Detecting fraudulent transactions quickly is essential to prevent financial losses and protect customers.

This project analyzes transaction data to identify patterns that may indicate fraudulent activity.

## Objectives
- Analyze financial transaction data
- Identify patterns associated with fraudulent transactions
- Visualize relationships between transaction attributes and fraud risk

## Technologies Used
- Python
- Pandas
- NumPy
- Matplotlib
- Seaborn

## Dataset
A simulated dataset of financial transactions containing:

- Transaction ID
- Customer ID
- Transaction Amount
- Transaction Time
- Location Risk Indicator
- Previous Fraud History
- Fraud Label

## Key Analysis
The analysis explores several factors that may influence fraud:

- Distribution of fraudulent vs legitimate transactions
- Relationship between transaction amount and fraud
- Fraud probability by transaction time
- Correlation between transaction attributes
- Fraud risk segmentation based on transaction amount

## Key Insights
- High-value transactions are more likely to be associated with fraud.
- Customers with previous fraud history show increased fraud risk.
- Transaction timing patterns can reveal suspicious activity.

## Applications
This type of analysis helps financial institutions build systems for:

- Fraud monitoring
- Risk detection
- Transaction verification

## Conclusion
The project demonstrates how transaction data can be analyzed to identify fraud indicators and support risk detection strategies in financial systems.
